Features

Everything you need for P2P chat

Powered by WebRTC DataChannel. No plugins, no extensions — just pure browser APIs.

End-to-End Encrypted

All traffic is encrypted via DTLS-SRTP. Your data never touches a third-party server.

100 KB Chunked Transfer

Files split into 100 KB chunks over RTCDataChannel. Reliable delivery with progress tracking.

Image Collage

Multiple images grouped into a beautiful collage layout — just like Telegram.

Rich Audio Player

Play/pause, seek, metadata display (title, thumbnail, duration). Works on all browsers — uses native <audio> element.

Multi-Attachment

Attach multiple images and audio files in a single message. Remove individual files before sending.

Real-time Transfer Progress

Live progress bars while sending / receiving files over the data channel.

How it Works

Connect in 3 steps

No accounts. No passwords. Just signal exchange.

1

Host creates an Offer

Click "Host a Session". An SDP offer code is generated — copy and share it with your peer (via any channel).

2

Peer creates an Answer

Peer opens "Join as Peer", pastes the offer code, and generates an answer code. Share that back with the host.

3

Direct P2P connection!

Host pastes the answer. WebRTC negotiation completes — you're now directly connected, no server required.

Security

Your privacy is the default

🔒

DTLS-SRTP

WebRTC's built-in mandatory encryption for all data.

🚫
